    Classification Accuracy Validation

    To better measure the accuracy of training result, we use the provided python scripts to obtain the error matrix and accuracy indexes including overall accuracy (OA),user's accuracy (UA) and producer's accuracy (PA)

  • Error Matrix
    also known as the confusion matrix, is used to compare reference data (true values) with classified data (predicted values). Rows represent classification results, and columns represent reference data. Each element in the matrix indicates the number of samples where the corresponding classification matches the reference value.
    Thematic Value known as the Hazard 1/2 value we set for No Landslide Zone & Landslide Inventory
    • ⏵ User's Accuracy (UA)
      indicates the proportion of samples in a particular classification category that are correctly classified, reflecting the reliability of the classification results. \(UA_i = \frac{n_{ii}}{n_{i+}} \)
    • \(n_{ii} \) is the number of samples where both the classification result and the reference value are i (diagonal element).
    • \(n_{i+} \) is the total number of samples classified as i (sum of the i-th row)
    ⏵ Producer's Accuracy (PA)
      indicates the proportion of samples in a particular reference category that are correctly classified, reflecting the effectiveness of the classifier. \( PA_i = \frac{n_{ii}}{n_{+i}} \)
    • \(n_{ii}\) is the number of samples where both the classification result and the reference value are i (diagonal element).
    • \(n_{+i}\) is the total number of samples with the reference value ii (sum of the i-th column).
    ⏵ Overall Accuracy (OA)
      indicates the proportion of all samples that are correctly classified, measuring the overall performance of the classification. \( OA = \frac{\sum_i n_{ii}}{\sum_i n_{i+}} \)
    • \({\sum_i n_{ii}} \)is the sum of all diagonal elements, representing the number of correctly classified samples.
    • \({\sum_i n_{i+}}\)is the total number of samples.

    Problems encontered

    1. NDVI mask
      To address the issue of missing NDVI data in our study area, we processed the clipped NDVI into vector data and performed a difference operation with the boundary vector data of the study area. The result is the vector data of the missing NDVI parts. We then merged this vector with the vector data of LS and NLZ areas and performed a difference operation to obtain the areas with available NDVI data. Within this area, we conducted random selection for creating the training and testing sets.
    2. Landslide inventory polygons cutting(splitting dataset manually for some specific conditions)
      Due to the presence of many disconnected polygons in the LS and NLZ data, selecting one polygon may cover an excessively large area, which could result in uneven distribution of the training and validation sets. Therefore, we manually divided these two vector datasets, splitting polygons with internal boundaries into smaller, uniformly distributed sections. After these divisions, we performed merging and other operations to ensure that the training/testing datasets were as evenly distributed as possible within the study area.
